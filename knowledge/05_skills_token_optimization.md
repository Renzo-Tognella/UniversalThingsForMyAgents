## Key Concepts & Techniques

### Token Optimization Strategies
- **Prompt Compression**: Using LLMLingua/LLMLingua-2 to remove non-essential tokens (up to 20x reduction)
- **Prompt Caching**: Caching repetitive prefix tokens to reduce cost by 90% and latency by 80%
- **KV Cache Optimization**: Sharing and offloading KV caches for multi-agent and long-context scenarios
- **Batch Processing**: Using Batch API for 50% cost reduction on non-real-time tasks
- **Context Compaction**: Automatically compressing conversation history to fit within token limits
- **Structured Outputs**: Using JSON schemas to constrain output and reduce waste tokens
- **Dynamic Compression Ratios**: Adjusting compression based on content importance
- **Semantic Compression**: Removing tokens based on information entropy rather than fixed ratios

### Agent Skills Design Patterns
- **Layered Context Architecture**: System, Task, Tool, and Memory layers for organized instructions
- **XML/Markdown Structuring**: Using structured formats for unambiguous prompt parsing
- **Few-shot Examples**: 3-5 examples with `<example>` tags for consistent output
- **Role Prompting**: Setting agent identity to focus behavior and tone
- **Tool Description Engineering**: Precise tool descriptions for correct agent tool selection
- **Error Handling Instructions**: Explicit patterns for graceful degradation
- **State Tracking**: Using structured formats (JSON) for agent state management
- **Verification Loops**: Self-check instructions to catch errors before completion

### Cost Reduction Techniques
- **Model Selection**: Choosing smaller models when possible (mini/nano variants)
- **Token Minimization**: Optimizing prompts for brevity while maintaining quality
- **Flex Processing**: Lower costs for non-production workloads
- **Prompt Template Reuse**: Caching common prompt prefixes
- **Adaptive Thinking**: Using effort parameters to control reasoning token spend
- **Predicted Outputs**: Specifying expected outputs to reduce generation cost
